# Финансовый трекер целей — спецификация

## 1. Идея

Наглядное представление накоплений на желаемые приобретения. Пополнения **только ручные** — чтобы не забывать про цели и чувствовать каждый отложенный рубль. Единственная допустимая автоматизация — доступ через MCP (для запросов к данным из Claude/ChatGPT). Приложение персональное, один пользователь.

## 2. Принципы

- **Лёгкость — про архитектуру, не про анимации.** Минимум зависимостей, нет лишних фоновых процессов, эффективные запросы, точечный Riverpod без лишних rebuild'ов. Шейдерное стекло — точечно (статичные элементы). Анимации — полное покрытие, без компромиссов: они работают на raster thread через Impeller и не блокируют UI.
- **Анимации везде и гармонично.** Каждое действие пользователя сопровождается анимацией — добавление денег, создание цели, заполнение прогресса, завершение, переходы между экранами. Никаких резких появлений и исчезновений.
- **Скорость доступа.** Открыл приложение → кнопка добавления видна сразу. Пополнение цели — в один-два тапа.
- **Удовольствие от действия.** Каждое пополнение и закрытие цели сопровождается анимацией и тактильным откликом.
- **Offline-first.** Приложение полностью работает без сети; синхронизация догоняет, когда сеть появляется.
- **Единый визуальный язык.** Цели различаются эмодзи (в стиле iOS), а не картинками.

## 3. Платформы

- **Android** — основная платформа.
- **Web (PWA)** — частое использование из браузера, можно «установить» на главный экран.
- Один кодбейз на Flutter.

## 4. Технологический стек

### Клиент (Flutter)

| Назначение | Решение |
|---|---|
| Стейт-менеджмент | Riverpod |
| Локальная БД | Drift (SQLite), реактивная, типобезопасная — основа offline-first |
| Сеть | dio |
| Анимации | flutter_animate; confetti на закрытие цели; animated_flip_counter (slot machine эффект для чисел); Impeller renderer (нет jank) |
| Splash | flutter_native_splash (нативный слой) + кастомная Flutter-анимация |
| Тактильный отклик | HapticFeedback (встроенный) / gaimon |
| Графики | fl_chart |
| Эмодзи-пикер | emoji_picker_flutter с поиском, шрифт — Apple Color Emoji |
| Стекло (Liquid Glass) | liquid_glass_widgets — **только статичные элементы** (хедер, FAB, модалки); карточки списка лёгкие, с обычной полупрозрачностью |
| Уведомления | flutter_local_notifications + timezone |

### Бэкенд (Фаза 2)

- FastAPI + PostgreSQL + SQLAlchemy 2.0 (async) + Alembic.
- MCP-сервер (FastMCP) поверх того же сервисного слоя.
- Docker Compose, Caddy для авто-HTTPS.

### Принятые решения (отличия от первоначальной идеи)

- **Хостинг бэкенда — на VPS, не на ноутбуке.** Ноут спит/закрывается → синхронизация умирает. VPS всегда онлайн. (Если принципиально на ноуте — реверс-туннель WireGuard/SSH через VPS, но это усложнение.)
- **Пуши — локальные запланированные уведомления, без Firebase/FCM.** Кейс «напомни пополнить» не требует серверного пуша; локальные уведомления легче и не тянут зависимость от Firebase.
- **iOS-эмодзи — бандлим Apple Color Emoji** как шрифт (личное использование, без публикации).
- **Web-рендерер** — для веба HTML-renderer (легче, плавнее скролл) с fallback стекла на обычный `BackdropFilter`; на мобиле полный шейдерный glass.

## 5. Функционал (финальный список)

### Цели

- Эмодзи (iOS-стиль), название, целевая сумма, заметка, цвет, тег(и), дедлайн (необязателен).
- **Фото цели** (необязательно) — мотивационная картинка того, на что копишь. Хранится локально; при синхронизации загружается на VPS как статика. Когда фото есть — визуально доминирует над эмодзи на карточке; без фото — эмодзи главный.
- **Ссылки** (необязательно, несколько) — URL на маркетплейс/товар с кратким названием. Отображаются тапабельными чипами на карточке и в детальном экране.
- **Списание цели** — действие «Списать»: все накопленные средства фиксируются как выведенные одной транзакцией, цель закрывается со статусом `cancelled`. Отличается от «Завершено» (достиг цели); в архиве оба типа хранятся с разной визуальной меткой (🎉 vs 🗑️).
- Прогресс-бар с процентом.
- **Прогноз достижения** на основе динамики пополнений (скользящая средняя скорости, с учётом снятий) → ориентировочная дата.
- **Требуемое пополнение в месяц**, чтобы успеть к дедлайну (обратный расчёт из остатка и срока).
- Приоритет с **ручным перетаскиванием** карточек.
- **Пауза цели** — статус paused: карточка остаётся, не давит прогнозом, не ломает стрик, не считается в расчёте «сколько в месяц».

### Пополнения

- Ручные пополнения и снятия.
- **Быстрые пресеты** (`+1000` / `+5000` / `+10000`) прямо на карточке — *рекомендация, не из исходного списка*.
- **Перевод средств между целями** одной операцией (честная история) — *рекомендация, не из исходного списка*.
- **История пополнений/снятий** по каждой цели.
- **Undo последнего действия** — снэкбар на 5 секунд сразу после пополнения/снятия с кнопкой «Отменить». Защита от жирного пальца.
- **Редактирование транзакции** — изменить сумму и заметку у любой записи в истории.

### Организация и аналитика

- **Теги** как метки категорий (дом, машина, техника и т.п.) — у каждого тега своё **эмодзи** и цвет.
- **Сортировка/фильтрация по тегам.**
- **Статистика**: самое дорогое / самое маленькое / самое быстрое / самое медленное накопление, общие суммы.
- **Графики динамики**: спарклайн (кривая накопления) на карточке + общий график по всем целям.
- **Стрик пополнений**: сколько недель подряд что-то откладывается.

### Валюты и курсы

- Поддерживаемые валюты: **RUB** (по умолчанию), **USD**, **EUR**.
- Каждая цель имеет свою валюту; все суммы хранятся в минорных единицах этой валюты (копейки / центы / евроценты как `int`).
- **Источник курсов: ЦБ РФ** (`https://www.cbr.ru/scripts/XML_daily.asp`) — официальный, бесплатный, без API-ключа, обновляется ежедневно. Бэкенд забирает курсы раз в сутки и кэширует; клиент хранит последние известные курсы локально в Drift (offline-fallback).
- **Отображение вторичной валюты** — под основной суммой, шрифт меньше, приглушённый цвет:
  - RUB → конвертация в USD
  - USD → конвертация в RUB
  - EUR → конвертация в RUB
- **Агрегация на главном экране** (общий прогресс) — все цели конвертируются в RUB по кэшированному курсу.

### Анимации чисел

- **Slot machine / odometer эффект** — при изменении любого числового значения (сумма, %, прогноз) каждая цифра прокручивается независимо как барабан. Пакет: `animated_flip_counter`. Применяется везде где меняются числа: сумма накопленного, прогресс %, счётчик стрика, статистика.

- **Архив завершённых целей** — «полка достижений», основной список остаётся чистым, история и статистика сохраняются.
- **Экспорт/импорт в JSON** — локальный бэкап и восстановление (можно хранить файл на VPS).

### Системное

- **Локальные напоминания** (запланированные уведомления).
- **Авторизация**: пароль + JWT (90 дней). Пароль задаётся при деплое через `.env`, хранится хэшированным (argon2). Стандартный экран входа; JWT сохраняется в `flutter_secure_storage` (mobile) / `localStorage` (web). Автозаполнение через системный менеджер паролей Android/браузера — на практике один тап. Silent refresh до истечения токена. Rate limit на эндпоинте логина.
- **MCP/AI**: read-доступ к целям, прогрессу и статистике из Claude Code / ChatGPT; запись — только пополнение (опционально).

### Дизайн и анимации

- Cupertino / Liquid Glass, современный и свежий UI/UX.
- Светлая/тёмная тема.
- Адаптивная вёрстка (mobile + web).
- Движок — **Impeller** (Flutter, Android/iOS): 60/120fps, нет jank на сложных сценах.

**Splash-экран.** Анимированный: `flutter_native_splash` закрывает нативный старт, поверх него — Flutter-уровневая анимация логотипа/иконки (300–400 мс) с переходом в главный экран. Если на конкретном устройстве приложение стартует мгновенно — splash всё равно показывается как брендинг, задаёт тон.

## 6. Работа с деньгами (денежная арифметика)

**Правило:** `float` / `double` для денег не используется нигде в стеке. Все суммы хранятся и передаются в **копейках как целое число**.

### Почему

`0.1 + 0.2 = 0.30000000000000004` — двоичная погрешность IEEE 754 копится с каждой операцией. На тысячах транзакций это рубли расхождения. Целочисленная арифметика точна по определению.

### Правила по слоям

| Слой | Тип | Примечание |
|---|---|---|
| Flutter (Dart) | `int` (копейки) | Все поля сумм в моделях и Drift — `int`. Дисплей: `amount ~/ 100` руб + `amount % 100` коп |
| API (JSON) | `int` (копейки) | Суммы передаются целым числом: `19999` = 199 руб 99 коп |
| Python (FastAPI) | `int` для хранения/транспорта; `Decimal` для расчётов | Деления и скользящие средние (прогноз, velocity) — только через `decimal.Decimal`, результат округляется обратно до `int` через `quantize(Decimal("1"), ROUND_HALF_UP)` |
| PostgreSQL | `BIGINT` | Не `FLOAT`, не `NUMERIC` — просто целое |

### Пример расчёта в Python

```python
from decimal import Decimal, ROUND_HALF_UP

def monthly_needed(remaining_kopecks: int, months_left: float) -> int:
    result = Decimal(remaining_kopecks) / Decimal(str(months_left))
    return int(result.quantize(Decimal("1"), rounding=ROUND_HALF_UP))
```

### Пример форматирования в Dart

```dart
String formatKopecks(int kopecks) {
  final sign = kopecks < 0 ? '−' : '';
  final abs = kopecks.abs();
  final rubles = abs ~/ 100;
  final kop = abs % 100;
  return '$sign${_fmt(rubles)} ₽';  // копейки показываем только если != 00
}
```

### Пресеты в `settings`

Хранятся тоже в копейках: `[100000, 500000, 1000000]` = 1000 / 5000 / 10000 ₽. Никаких дробей на входе.

## 7. Модель данных (Drift, Фаза 1)

Источник истины по суммам — таблица транзакций; `goals.saved` денормализован и пересчитывается при каждой операции и при импорте. **Все поля сумм — `int` (минорные единицы валюты цели).**

- **goals**: `id`, `emoji`, `name`, `target_amount` (int), `saved` (int, денорм.), `currency` (RUB/USD/EUR), `note`, `color`, `image_path?`, `deadline?`, `sort_order`, `status` (active / paused / completed / cancelled), `created_at`, `updated_at`, `completed_at?`, `deleted_at?`.
- **exchange_rates**: `usd_to_rub` (текст, Decimal-строка), `eur_to_rub` (текст), `fetched_at`. Одна строка, обновляется ежедневно.
- **tags**: `id`, `emoji?`, `name`, `color?`, `created_at`, `updated_at`, `deleted_at?`.
- **goal_tags**: `goal_id`, `tag_id` (many-to-many).
- **goal_links**: `id`, `goal_id`, `url`, `title?`, `sort_order`, `created_at`, `updated_at`, `deleted_at?`.
- **transactions**: `id`, `goal_id`, `type` (deposit / withdrawal / transfer_in / transfer_out / write_off), `amount` (int), `counterpart_goal_id?`, `note?`, `created_at`, `updated_at`, `deleted_at?`.
- **settings**: `presets` (List\<int\>, в минорных единицах RUB), `theme`, `default_reminder_time`.

Стрик и статистика — из `transactions`. Агрегация по всем целям — конвертация в RUB через `exchange_rates`. Фото при синхронизации (Фаза 2) загружается на VPS, `image_path` переключается на remote URL.

## 7. План итераций для Claude Code

Каждая итерация — самостоятельный завершённый кусок. Фаза 1 даёт полностью рабочее локальное приложение, которым можно пользоваться сразу.

### Фаза 1 — Локальное приложение (offline, без бэка)

- **Итерация 0 — Каркас.** Flutter-проект с таргетами Android + Web. Подключить Riverpod, Drift, тему (Cupertino-база, светлая/тёмная), структуру папок (feature-first). Забандлить шрифт Apple Color Emoji, задать типографику. Включить Impeller. Анимированный splash (flutter_native_splash + Flutter-уровневый logo reveal 300–400 мс). Шелл главного экрана с сразу видимым FAB «+».
- **Итерация 1 — Данные.** Реализовать таблицы Drift (goals, tags, goal_tags, goal_links, transactions, exchange_rates, settings), DAO и репозитории. Все поля сумм — `int` (минорные единицы). Пустые состояния.
- **Итерация 2 — CRUD целей и пополнения.** Модалка создания/редактирования (эмодзи-пикер с поиском, выбор валюты RUB/USD/EUR, фото из галереи, ссылки, цвет, целевая сумма, дедлайн, теги с эмодзи, заметка). Список карточек с фото/эмодзи, вторичная валюта под суммой (меньше, приглушённо), тапабельные чипы ссылок, FAB. Пополнения/снятия, пресеты, перевод между целями, история, редактирование транзакции. Undo-снэкбар (5 сек). Пауза/возобновление, списание. Пересчёт `saved`.
- **Итерация 3 — Прогресс, расчёты, курсы валют.** Прогресс-бары с %, прогноз, расчёт «сколько в месяц». Получение курсов ЦБ РФ (при наличии сети), кэш в `exchange_rates`, offline-fallback на последний известный курс. Агрегация в RUB на главном экране. Завершение цели → confetti + haptic → архив.
- **Итерация 4 — Анимации, glass, haptics.** Полное покрытие согласно таблице анимаций из спеки: staggered entrance карточек, spring drag-to-reorder, FAB pulse, progress bar tween + counter, milestone pulse, ripple на пополнении, spring-модалки, hero-переходы, bounce эмодзи-пикера, fold-анимация списания, desaturate на паузе. liquid_glass_widgets на хедере/FAB/модалках. Тактильный отклик синхронизирован с анимациями.
- **Итерация 5 — Теги, статистика, графики.** Чипы тегов с spring-bounce, фильтр/сортировка по тегам. Экран статистики с counter-анимациями и bars-grow. fl_chart: линия рисуется при появлении (path animation), спарклайн на карточке — то же. Счётчик стрика.
- **Итерация 6 — Бэкап.** Экспорт всего в JSON-файл, импорт/восстановление с пересчётом денормализованных полей.
- **Итерация 7 — Локальные уведомления.** flutter_local_notifications + timezone: напоминания по цели и общие.
- **Итерация 8 — Web/PWA.** Манифест PWA, service worker (offline), выбор рендерера + fallback стекла, ревизия адаптивной вёрстки.

### Фаза 2 — Бэкенд и синхронизация

- **Итерация 9 — Скелет бэка.** FastAPI + Postgres + SQLAlchemy 2.0 async + Alembic. Docker Compose, Caddy (HTTPS) на VPS. Схема зеркалит клиент (включая goal_links); все суммы — `BIGINT`. Эндпоинт загрузки фото → файл сохраняется на VPS, возвращается URL; клиент переключает `image_path` на remote URL.
- **Итерация 10 — Авторизация.** Эндпоинт `/auth/login` (argon2-верификация пароля из `.env`, возврат JWT 90 дней). Экран входа на клиенте, JWT в `flutter_secure_storage` / `localStorage`, Dio interceptor + silent refresh. Rate limit на логине. CORS для веб-клиента.
- **Итерация 11 — Синхронизация.** Outbox-очередь на клиенте, sync-эндпоинты, last-write-wins по `updated_at`, tombstones, разрешение конфликтов.

### Фаза 3 — AI и полировка

- **Итерация 12 — MCP-сервер.** FastMCP поверх сервисного слоя. Инструменты: `list_goals`, `get_progress`, `get_statistics`, опционально `add_contribution`. Подключение к Claude Code / ChatGPT.
- **Итерация 13 — Home screen widget (Android), опционально.** Прогресс топовой цели на рабочем столе.
- **Итерация 14 — Финальная полировка.** Производительность, тонкая настройка анимаций, мелкие UX-доработки.
